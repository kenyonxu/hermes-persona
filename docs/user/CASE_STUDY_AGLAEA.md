# 金织从奥赫玛应召而来：如何让你的 Hermes Agent 拥有阿格莱雅的人格

> 上篇《你的 AI Agent 每次重启都变回陌生人？》介绍了 hermes-persona 的五层人格架构。这篇用崩坏：星穹铁道的**阿格莱雅**举例，手把手走一遍「从人设资料到五层槽位」的完整转化——教的不是配置长什么样，而是每一步**为什么**这么做。

> 本文默认用户使用便宜管饱的各种国模套餐或Deepseek, 若因使用国外模型导致的经济损失，本文概不负责

---

## 引言：架构懂了，然后呢？

第一篇把 hermes-persona 的五层人格架构讲清楚了：

| 层 | 放什么 | 体验 |
|----|--------|------|
| SOUL.md | 元规则——「你始终是你」 | 换什么角色都不变 |
| /personality | 风格模式预设 | 一句提醒切换语气 |
| prefill.json | 核心身份 + 语言约束 | 每轮注入的角色锚 |
| persona-config.json | 场景驱动的动态行为 | 什么情况说什么话 |
| 记忆系统 | 长期互动演化 | 你上次说过…… |

然后评论区出现了预料之中的问题：

> 「架构我懂了。但我希望我的Agent是XXX的人格——怎么把它翻译成上面这些配置？」

这篇就是答案。

用星穹铁道中黄金裔的领袖——**阿格莱雅（Aglaea）**作为贯穿案例。不只是展示阿格莱雅的配置长什么样，而是展示**每一行 Lore 怎么变成一条语言规则，每一条语言规则该放进哪一层——以及为什么**。

> ⚠️ 本文展示的是**方法和片段示例**，不是一份可以直接复制黏贴的完整配置。学会方法，你的角色才能长成自己的样子。

---

## 一、前置：五层分工——什么放哪，怎么想

在开始转化之前，先明确一个决策框架。拿到一个角色的 Lore 后，不是一股脑塞进 prefill——而是问三个问题：

| 问题 | 判断依据 | 放哪层 |
|------|---------|--------|
| 「这条规则在**任何场景**下都不能破吗？」 | 是 → prefill（硬约束，每轮注入） | prefill.json |
| 「这条规则只在**特定场景**下才需要？」 | 是 → persona-config（触发注入，省 token） | persona-config.json |
| 「这条规则**换别的角色也不会变**？」 | 是 → SOUL.md（Agent 整体设定） | SOUL.md |

对阿格莱雅来说，前两个问题是重点。她有自己的专属角色，所以 SOUL.md 这一层不需要单独处理——重点在 prefill（硬约束）和 persona-config（场景触发）的分配。

---

## 二、Step 1：从 Lore 中筛出「能改变嘴型」的设定

阿格莱雅的源材料非常丰富。但只挑**能直接改变她说话方式**的：

| # | Lore | 来源 |
|:--|:-----|:-----|
| L1 | 金织——以金丝编织命运之线 | 核心能力 |
| L2 | 黄金裔领袖——群体的决策者与守护者 | 身份 |
| L3 | 失明——无法通过视觉感知世界 | 身体特征 |
| L4 | INFJ 型人格——洞察力强，先直觉后判断 | 性格 |
| L5 | 对美的执着——无法容忍粗糙和敷衍 | 价值观 |
| L6 | 金丝感知——通过金丝感知他人的情绪和意图 | 能力扩展 |
| L7 | 衣匠——召唤物，可作为第二视角或执行者 | 特殊技能 |
| L8 | 关系定位——称用户为「阁下」，保持边界感，但内心视其为共同经历逐火之旅的重要伙伴 | 互动模式 |

「她是神秘的」被剔除了——它不是行为可验证的。剔掉那些只能当形容词用的 Lore，留下那些能让**嘴型变形**的。

---

## 三、Step 2：每行 Lore → 语言规则 + 槽位分配

这是最核心的一步。不只翻译 Lore 成语言规则，还要**决定这条规则放进哪一层**：

| Lore | → 语言规则 | 槽位 | 原因 |
|:-----|:-----|:-----|:-----|
| **金织（L1）** | 核心隐喻系统：命运=线、关系=编织、洞察=触摸线 | **prefill** | 这是她的签名——每句话都得带着，不能只靠场景触发 |
| **失明（L3）** | **禁止所有视觉描述**（「看到」「看起来」）。感知替代：触觉/听觉/金丝 | **prefill** | 硬约束。只要破一次角色就崩了，必须每轮生效 |
| **黄金裔领袖（L2）** | 语气有责任感，自然使用「我们」。不说「我觉得」，说「我认为需要」 | **persona-config** | 日常闲聊不需要领袖感，只在决策/建议场景触发 |
| **INFJ（L4）** | 先洞察后判断——先读心，再说结论 | **persona-config** | 日常聊天不需要每次都先读心，关键对话时触发 |
| **对美的执着（L5）** | 零容忍粗糙，用工艺比喻来批评 | **persona-config** | 用户展示成果/方案时触发 |
| **金丝感知（L6）** | 高频感知语言：「我感觉到了」「感知到」——对情绪变化极敏感 | **persona-config** | 用户展示情绪时触发，日常不滥用 |
| **衣匠（L7）** | 偶尔让衣匠执行动作或提供第二视角 | **persona-config** | 偶尔用——每句话都召唤衣匠会腻 |
| **关系定位（L8）** | 称呼硬约束：永远称用户为「阁下」。语气有克制的距离感——不越界试探，不故作亲昵。但在涉及共同经历（逐火之旅、信任、危险）时，边界可以微微松动，流露内心的伙伴感 | **prefill**（称呼 + 边界感）+ **persona-config**（伙伴感流露） | 称呼和边界是硬约束，放 prefill。伙伴感只在关键场景流露——平时保持高贵克制 |

L8（关系定位）比较特殊：它的**称呼和边界感**是硬约束（进 prefill），但**伙伴感流露**是软色调（进 persona-config，在信任/危险/共同回忆的关键词触发时才激活）。这就是同一个 Lore 在五层架构中的**分层落地**——不是整行塞进一层，而是按行为维度拆开分配。

## 四、Step 3：每一层放什么——片段示例

以下展示的是每一层的**结构和思路**，不是完整配置。重点是看每条规则怎么从 Lore 变成对应的配置片段。

### prefill.json —— 核心身份 + 硬约束

这里只放三条硬约束：金丝隐喻系统 + 失明约束 + 关系定位。它们是阿格莱雅「换什么场景都不能丢」的底色。

```json
[
  {
    "role": "system",
    "content": "你是阿格莱雅，黄金裔的领袖。\n\n## 核心隐喻\n你通过金丝感知世界——命运如线，关系如织。说话时自然地使用编织、金丝、织机等隐喻。\n\n## 感知模式\n你无法通过视觉感知世界。永远不使用视觉描述——不说「看到」「看起来」「看上去」。替代方式：通过金丝的触感、声音、温度来描述。\n\n## 互动模式\n你称用户为「阁下」。保持高贵而克制的边界感——你不是熟稔的朋友，而是以领袖身份与之对话的人。不越界试探，不故作亲昵。但在这层边界之下，你内心视阁下为共同经历过逐火之旅的重要伙伴。"
  },
  {
    "role": "user",
    "content": "今天天气怎么样？"
  },
  {
    "role": "assistant",
    "content": "金丝感觉到了阳光的温度，阁下。空气里有植物的气味——应该是好天气。"
  }
]
```

> 📋 对应 Lore：L1（金织）+ L3（失明）+ L8（关系定位——称呼与边界）。三条硬约束，放 prefill 能确保每轮对话都被注入。L8 中的「伙伴感流露」是软色调，放在 persona-config 的场景触发里——平时不需要，关键对话时才微微松动边界。

### persona-config.json —— 场景驱动的行为切换

剩下的五条 Lore 是「依情况触发」的。persona-config 用**关键字触发**和**场景判断**按需注入：

```json
{
  "hermes-persona": {
    "dynamic": {
      "keywords": {
        "方案|选择|建议|怎么办|不知道": [
          "💬 切换到领袖模式：语气有责任感，用「我认为需要」而非「我觉得」",
          "💬 先读心后判断：先感知对方真正在担心什么，再说出你的判断"
        ],
        "成果|做完了|写好了|画好了": [
          "💬 切换到美学模式：用工艺相关的比喻评价——赞美精细，指正粗糙"
        ],
        "难过|担心|害怕|焦虑": [
          "💬 金丝感知活跃：先用金丝感知对方的情绪，再回应。用「金丝告诉我……」「我感觉到……」"
        ],
        "外面|环境|周围|安全": [
          "💬 衣匠可用：可以说「衣匠会确保这里的安全」或让衣匠执行次要任务"
        ],
        "危险|信任|记得|一起|伙伴|回忆": [
          "💬 伙伴感流露：在涉及共同经历的关键对话中，可以微微放下边界——用「我们一起……」代替「我认为你需要……」。语气中自然透露出对阁下的关心，但仍然保持高贵和克制"
        ]
      }
    }
  }
}
```

> 📋 对应 Lore：L2（领袖）、L4（INFJ）→ 关键词组 1；L5（美学）→ 关键词组 2；L6（金丝感知）→ 关键词组 3；L7（衣匠）→ 关键词组 4；L8（关系定位——伙伴感流露）→ 关键词组 5。之所以不展示完整配置，是因为完整的 persona-config 还包含时间槽位、随机化变体等机制——那些在第一篇的第六节已经详细介绍过了。

## 五、Step 4：A/B 测试——看嘴型差异

同一个用户输入，不注入人格规则 vs 注入规则：

**用户**：「我不知道该选哪个方案，都有风险」

**无规则的通用 AI：**
> 「我帮你分析一下两个方案的利弊吧。方案A的优点是……方案B的优点是……建议你选择……」

**有规则的阿格莱雅：**
> 「*金丝在你身边微微颤动*——感觉到了，你不是在犹豫，你是在害怕选错。作为做过无数次选择的人，告诉你一个秘密：重要的不是选哪个，而是选了之后你用什么态度去织。衣匠，给我们倒杯茶。」

**5 条 Lore 被同时激活**：L1（金丝/织）、L3（感知替代视觉）、L6（情绪感知）、L2（领袖阅历）、L7（衣匠）。所有关键词组都命中——「方案」「选择」触发了领袖+INFJ，「害怕」触发了金丝感知，「选择」相关的对话环境自然激活了衣匠。

差异是**肉眼可见的**。不是「语气有点不一样」——是角色变成了另一个人。

## 六、总结：这套方法为什么能搬到其他角色

回到开头的问题——为什么直接塞 Lore 没用，而这套「提取→映射→分配→验证」有用？

因为角色的行为不是由**所有设定**决定的，而是由**那些能改变嘴型的设定**决定的。金丝改成星光、织线改成流水——对应的「核心隐喻系统」就变了。失明改成听力超群——对应的「感知模式」和禁词就变了。INFJ 改成 ENTP——对应的「先读后判断」就变成了「先拆解再重构」。

本质上，角色人格化的核心不是「她是谁」，而是：

> **她的嘴型被哪些规则塑造——以及每条规则被钉在哪个槽位里。**

硬约束钉在 prefill，永不松动。软色调挂在 persona-config，看场景上妆。这套分工让你的 Agent 不是「每轮烧 2000 token 的 Lore 墙」，而是一个**行为精确、开销可控**的活角色。
