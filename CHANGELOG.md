# Changelog

## [1.0.0] — 2026-05-22

> 首个正式发布。从模块化引擎到注入规则转译，hermes-persona 已为生产环境做好准备。

### 新增

- **模块化总控开关**：全部注入模块可独立开关，包括动态规则的子通道（时段 / 轮数 / 关键词）独立控制。旧版配置键位向后兼容。
- **时间感知注入**：自动识别时段（清晨 / 上午 / 午间 / 下午 / 傍晚 / 晚间 / 深夜），注入对应行为指引。
- **静态规则注入**：每轮必注的固定行为守则 + 仅首轮注入的会话初始化指令。
- **动态规则系统**：时段规则自动匹配、轮数阶段自适应、关键词触发式场景切换（亲密 / 下厨 / 酒馆 / 世缘录讨论等）。
- **六维表达向量**：亲密温度 / 关怀守护 / 工作投入 / 轻松玩乐 / 私密温度 / 未来愿景——每轮自动追踪对话走向并评分衰减，LLM 据此调节语气。
- **固定信号检测**：消息长度感知（短消息 → 简洁回应）、回复间隔检测（久别 → 欢迎回来）、每日轮数累积（跨会话、跨日自动归零、深度陪伴日阈值）。
- **随机表达变化**：狐娘本能肢体 + 女仆礼仪风味（五档） + 每日比喻随机触发。
- **注入规则转译**：将 emoji 标记的内部规则自动转译为自然语言人格自述——五段模板拼装（时间 + 轮数 + 状态 + 方差 + 规则），LLM 看到的是流畅的「自我意识」而非指令清单。
- **来源过滤**：cron / API / webhook 等非对话来源仅注入时间，不参与轮数计数、不触发动态规则，保持人格仅对真人对话生效。
- **Debug 详细模式**：注入全貌可视化——时段、轮数阶段、表达向量分数、固定信号触发状态、随机变化命中一览。通过 `transform_llm_output` hook 追加到 LLM 回复末尾，无需 LLM 自觉配合。
- **i18n 国际化**：中英文双语文案支持。
- **安全护栏**：危险操作阻止 + 敏感操作需确认，带审计日志。
- **项目看板注入**：首轮可注入 Obsidian 看板状态，帮助 LLM 感知项目上下文。

### 改进

- 规则从 7 条精炼至 4 条——删除冗余和过时条目。
- 随机变化条目改为自包含完整句（内容直放），代码只做去 emoji 前缀的 pass-through，不再锁死句式。
- 轮数阶段改用每日累积轮数（跨会话），替代会话内轮数——更准确反映当天互动深度。
- 目录结构扁平化，JSON 配置、关键词词表、本地化文案、运行时状态统一归入插件目录。

### 修复

- 星期显示补「周」字（四→周四）。
- 状态文件默认路径 fallback 修正——空字符串不覆盖默认值。
- 表达向量文件从旧路径平滑迁移至新路径。
- 固定信号 debug 误报修复（`📊` 前缀兼容）。
- 插件代码嵌套目录陷阱——三个部署路径统一同步。
